Restful API In ASP.NET: Introduction of REST &amp; Web API

<div id="div2" class="PaddingLeft5"><p align="justify"></p><p>In the previous article <a href="http://www.c-sharpcorner.com/UploadFile/4b0136/restful-api-in-Asp-Net-getting-started/"> <span style="color: #0000ff;">Getting Started in Web API</span></a>, we have learned the basic infrastructure of API and multiple clients which can consume API. Now, we will discuss here the architecture of Restful API and we will also get an overview of ASP.NET Web API introduction.<br> <br> <strong>What is REST<br> <br> </strong>REST is the acronym that stands for: Representational State Transfer. REST is an architectural style of distributed system. It is based upon the set of principles that describes how network resources are defined and addressed. These set of principles was first described by “Roy Fielding” in 2000. REST is bigger than Web Services. <br> <br>RESTful services uses HTTP (Hyper Text Transfer Protocol) to communicate. REST system interface with external systems as web resources identified by URIs (Uniform Resource Identifiers). <br> <br> <strong>Constraints of REST<br> <br> </strong>A Restful system should follow the constrains so it can be called Restful. There are 5 and 1 optional constraints of REST.<br> <br> <img src="http://csharpcorner.mindcrackerinc.netdna-cdn.com/UploadFile/4b0136/restful-api-in-Asp-Net-introduction-of-rest-web-api/Images/api.jpg" alt="api"><br> <br> <strong>Http Verbs<br> <br> </strong>Http verbs plays a very important role in the Restful Web API. The most common Http verbs are GET, PUT, POST and DELETE and these correspond to CRUD (Create, Read, Update and Delete) operations respectively.<br> <br>The following Http Verbs exists:</p> <ul> <li>Get</li> <li>Post</li> <li>Put</li> <li>Delete</li> <li>Trace</li> <li>Options</li> <li>Connect</li> <li>Patch</li> </ul> <p>I will discuss here the most common used verbs and we will also see that how we can use these verbs in my next articles in which we will perform database operations with the help of the above Http verbs.<br> <br>So, the following are the description of most common used Http verbs:<br> <br> <strong>GET<br> </strong><br>As the name specifies, this verb is used to retrieve the data or information. It does not have any other effect except getting data. <br> <br> <strong>Syntax<br> </strong><br> <em>[HttpGet]<br> </em><br>We can use the above syntax to define the method as a GET method. If we are retrieving the data from the database or displaying the data, then we can use this attribute to get the data. <br> <br> <strong>Use<br> </strong><br>Let’s see an example of a GET method:<br></p><div class="dp-highlighter"><div class="bar"></div><ol start="1" class="dp-c"><li class="alt"><span><span>[HttpGet,&nbsp;ActionName(</span><span class="string">"GetInfo"</span><span>)]&nbsp;&nbsp;</span></span></li><li class=""><span>public HttpResponseMessage&nbsp;GetInfo()&nbsp;&nbsp;</span></li><li class="alt"><span>{&nbsp;&nbsp;</span></li><li class=""><span>&nbsp;&nbsp;&nbsp;&nbsp;<span class="comment">//functional&nbsp;requirements</span><span>&nbsp;&nbsp;</span></span></li><li class="alt"><span>}&nbsp;&nbsp;</span></li></ol></div>In the above example there is a method named GetInfo which is defined as GET method. ActionName attribute is used to define the name of an action method.<br> <br> <strong>Call<br> </strong><br> <em>http://localhost:56828/api/sample/GetInfo<br> </em><br>By the help of above Requested URI, we can get the information. Sample is the controller name.<br> <br> <strong>POST<br> </strong><br>This verb is used to generate or create the resource. For example, If we have to add some information to the database then we must define the method as a POST method. The most common form to submit the data into the POST method is to pass the data into any entity object.<br> <br> <strong>Syntax<br> </strong><br> <em>[HttpPost]<br> </em><br>We can use the above syntax to define the method as a POST method. <br> <br> <strong>Use<br> </strong><div class="dp-highlighter"><div class="bar"></div><ol start="1" class="dp-c"><li class="alt"><span><span>[HttpPost,&nbsp;ActionName(</span><span class="string">"PostInfo"</span><span>)]&nbsp;&nbsp;</span></span></li><li class=""><span>public HttpResponseMessage&nbsp;PostInfo(Result&nbsp;result)&nbsp;&nbsp;&nbsp;</span></li><li class="alt"><span>{&nbsp;&nbsp;</span></li><li class=""><span>&nbsp;&nbsp;&nbsp;&nbsp;<span class="comment">//functional&nbsp;requirements</span><span>&nbsp;&nbsp;</span></span></li><li class="alt"><span>}&nbsp;&nbsp;</span></li></ol></div>In the above example there is a method named PostInfo which is defined as POST method. We have to assign the data to the properties of the entity (Result class).<br> <br> <strong>PUT<br> </strong><br>This verb is used to update the existing resource. For example, If we have to update some information to the database then we can define the method as a PUT method. We can send the data in the form of object as well as in a parameter too.<br> <br> <strong>Syntax<br> </strong><br> <em>[HttpPut]<br> </em><br>We can use the above syntax to define the method as a PUT method. <br> <br><strong>Use<br> </strong><div class="dp-highlighter"><div class="bar"></div><ol start="1" class="dp-c"><li class="alt"><span><span>[HttpPut,&nbsp;ActionName(</span><span class="string">"UpdateInfo"</span><span>)]&nbsp;&nbsp;</span></span></li><li class=""><span>public HttpResponseMessage&nbsp;UpdateInfo(Result&nbsp;result)&nbsp;&nbsp;</span></li><li class="alt"><span>{&nbsp;&nbsp;</span></li><li class=""><span>&nbsp;&nbsp;&nbsp;&nbsp;<span class="comment">//functional&nbsp;requirements</span><span>&nbsp;&nbsp;</span></span></li><li class="alt"><span>}&nbsp;&nbsp;</span></li></ol></div>In the above example there is a method as named UpdateInfo which is defined as PUT method. We can assign the id to update the info.<br> <br> <strong>DELETE<br> </strong><br>This verb is used to delete the existing resource. For example, If we have to delete some information to the database then we can define the method as a DELETE method. We can send the data in the form of object as well as in a parameter too.<br> <br> <strong>Syntax<br> </strong><br> <em>[HttpDelete]<br> </em><br>We can use the above syntax to define the method as a DELETE method.<br> <br> <strong>Use<br> </strong><div class="dp-highlighter"><div class="bar"></div><ol start="1" class="dp-c"><li class="alt"><span><span>[HttpDelete,&nbsp;ActionName(</span><span class="string">"DeleteInfo"</span><span>)]&nbsp;&nbsp;</span></span></li><li class=""><span>public HttpResponseMessage&nbsp;DeleteInfo(<span class="keyword">int</span><span>&nbsp;UserId)&nbsp;&nbsp;</span></span></li><li class="alt"><span>{&nbsp;&nbsp;</span></li><li class=""><span>&nbsp;&nbsp;&nbsp;&nbsp;<span class="comment">//functional&nbsp;requirements</span><span>&nbsp;&nbsp;</span></span></li><li class="alt"><span>}&nbsp;&nbsp;</span></li></ol></div>In the above example there is a method named DeleteInfo which is defined as DELETE method.
